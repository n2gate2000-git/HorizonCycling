# 開発完了報告書 - HorizonCyclingBridge

『Forza Horizon 6』とスマートローラー（FTMS対応デバイス）を双方向で中継し、ペダリングによるアクセル・ブレーキ操作と、ゲーム内の地形（勾配）に連動したスマートローラーの自動負荷調整を両立する、C# / .NET 8.0 ベースの Windows 用中継アプリケーションを実装しました。

---

## 1. 実装されたコードベースと役割

将来的なUI実装や保守性を考慮し、仕様書で提示されたアーキテクチャに準拠したクリーンなモジュール分割を行いました。

## 2. コア設計と実装の特長

1. **3次物理方程式の数値解析（ニュートン・ラフソン法）**:
   `SimulationMappingStrategy.cs` において、ペダリングパワー、空気抵抗、転がり抵抗、そしてゲームから取得した道路斜度（ピッチ角から算出）の4要素が釣り合う自転車の目標速度を、ニュートン法を用いて毎フレーム高速かつ正確に近似計算します。
   > [!TIP]
   > この物理モデルにより、**「ペダルを漕がなくても、下り坂であれば重力によって自転車が滑走して車が勝手に加速する」**という極めてリアルな実走感が物理計算だけで再現されます。

2. **アンチワインドアップ対策付きPID制御**:
   `SpeedPidController.cs` では、壁に激突して車が停止しているのにペダルを必死に回し続けているシチュエーションなどで、誤差の積分項が無限に巨大化して車が急発進するのを防ぐため、積分飽和をガードする制限ロジックを実装しています。

3. **WinRT BLEによる自動スキャンと接続**:
   `FtmsClient.cs` は `BluetoothLEAdvertisementWatcher` を使用し、スマートローラーが電源ONになると自動的にFTMS（サービスUUID: `0x1826`）を検知してバックグラウンドでペアリングし、自動接続を開始します。

4. **vJoyへの軽量P/Invokeバインディング**:
   外部の community DLL に依存することによる互換性の低下を避けるため、`vJoyInterface.dll` と直接 P/Invoke（DllImport）で通信する独自の `VJoyVehicleController` を実装し、無駄のない軽量動作を実現しました。

## 4. アプリケーションの実行手順

本システムを実機で動作させるための手順は以下の通りです。

### 準備事項
1. **vJoyドライバーのインストール**: PCに [vJoy](https://github.com/jshafer817/vJoy) がインストールされ、仮想ジョイスティックが1台以上有効になっていること。
2. **DLLの配置**: ビルド先のフォルダ（`bin\Debug\net8.0-windows10.0.19041.0\`）に `vJoyInterface.dll`（64bit版）が配置されていること。
3. **ゲーム側の設定**: 『Forza Horizon 6』の「HUDとゲームプレイ」設定で **UDPデータ出力を有効** にし、IP「`127.0.0.1`」、ポート「`5000`」を指定します。ゲーム内のコントローラー設定で、vJoyデバイスの入力（Axis X/Y）をアクセル・ブレーキとして割り当てます。

### 起動と操作
1. コンソールでプロジェクトディレクトリに移動し、**`dotnet run`** を実行します。
2. 起動時にモードを選択します。
   - `1`：アーケードモード（とにかくペダルを回した分だけスーパーカーがかっ飛ばす）
   - `2`：シミュレーションモード（現実の自転車と全く同じ速度で車が走るリアルライド）
3. スマートローラーの電源をONにします。アプリが自動検知してBLE接続を行います。
4. Forzaでドライブを開始すると、ペダリングパワーに応じて車が加速し、登り坂や下り坂に入るとスマートローラーの負荷が自動的に変化します。
5. 終了するには、コンソール上で任意のキーを押すと、BLEとUDPを安全に切断・クリーンアップして終了します。

### vJoyのインストール

vJoyのインストール中に再起動ループに陥ったり、セットアップが途中でフリーズしたりするのは、Windows 10やWindows 11環境で**非常によくあるトラブル**です。

これは主に、オリジナルのvJoy（Shaul氏が開発したもの）が長らく更新されておらず、現在のWindowsの厳格な「ドライバ署名要件」に引っかかってしまい、ドライバの登録処理でインストーラーが無限待ち状態になってしまうことが原因です。

この状態から抜け出し、正しくインストールするための手順をまとめました。

### 1. フリーズしたセットアップの強制終了

まずは止まってしまったインストーラーを終了させます。

1. `Ctrl` + `Shift` + `Esc` キーを押して**タスクマネージャー**を開きます。
2. プロセス一覧から「vJoy Setup」や関連するインストーラーのタスクを見つけ、右クリックして「タスクの終了」を選択します。

### 2. 中途半端なインストールのクリーンアップ

失敗した状態のファイルを綺麗に削除してリセットします。

1. **アプリの削除:** Windowsの「設定」 ＞ 「アプリ」 ＞ 「インストールされているアプリ」を開き、一覧に「vJoy」があればアンインストールします。
2. **デバイスマネージャーの確認:** スタートボタンを右クリックし、「デバイス マネージャー」を開きます。
* 「ヒューマン インターフェイス デバイス」の中に「vJoy Device」や、黄色い警告マーク（！）がついている不明なデバイスがあれば、右クリックして「デバイスのアンインストール」を選択します。


3. **残存フォルダの削除:** エクスプローラーを開き、`C:\Program Files\vJoy` フォルダが存在していれば、フォルダごと削除します。
4. ここで一度PCを**再起動**します。（再起動後にまたウィザードが勝手に立ち上がった場合は、キャンセルして閉じてください）

### 3. 最新のWindowsに対応した「フォーク版」を使用する

公式サイト（vjoystick.sourceforge.net）にある古いバージョン（v2.1.8やv2.1.9）は現在のWindowsでは正常にインストールできません。有志のエンジニアがWindows 10/11向けにドライバ署名問題を修正したフォーク版（派生版）を使用する必要があります。

1. GitHubなどの有志リポジトリから最新の対応版をダウンロードします。
* 現在最も安定しているとされるのは、jshafer817氏によるvJoyフォーク（v2.1.9.1）や、njz3氏によるvJoyフォーク（v2.2.1.1など）です。
* ブラウザで「vJoy GitHub releases jshafer817」などで検索し、最新の `.exe` インストーラーをダウンロードしてください。



### 4. 管理者権限での再インストール

1. ダウンロードした新しいインストーラーを**右クリック**します。
2. 必ず「管理者として実行」を選択して起動してください。
3. インストールウィザードを進めます。（コンパニオンアプリなどのチェックボックスがあり、特に不要であればチェックを外しておくとより確実です）

この手順を踏むことで、途中で止まることなくドライバが正しく認識され、インストールが完了するはずです。
