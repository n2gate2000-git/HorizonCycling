# 各種センサーを用いた固定ローラーでの活用方法の検討

本ドキュメントでは、スマートローラー（インタラクティブトレーナー）を持たないユーザーが、各種センサー（パワーメーター、スピードセンサー、ケイデンスセンサー）と固定ローラー（非連動型ローラー）を組み合わせて、Forza Horizon 6 の車両をコントロールする「HorizonCyclingBridge」を利用するための仕様を検討します。

## 1. 固定ローラー利用時のシステム概要とテレメトリの活用

現状の HorizonCyclingBridge は、以下の2つのループで成り立っています。
1. **入力ループ**: ペダリングパワー ➔ ゲーム内の車のアクセル制御
2. **フィードバックループ**: ゲーム内の斜度・テレメトリ ➔ スマートローラーの負荷抵抗

固定ローラーを利用する場合、2つ目の「ハードウェアへのフィードバック（自動負荷調整）」は物理的に不可能です。ペダルの重さは常に一定（または手動調整）となります。

しかし、**ゲーム内からのテレメトリデータ（特に勾配データ）は無駄にはならず、シミュレーションモードにおいてフル活用されます。** 
本システムの「シミュレーションモード」は、センサーから取得した「パワー（W）」と、テレメトリから取得した「現在の道路勾配（%）」を自転車の物理方程式に入力し、目標速度を計算しています。
そのため、ローラー側の物理的な負荷が変わらなくても、以下のようなリアルな連動体験が可能です。

- **上り坂**: 同じパワー（W）で漕ぎ続けても、テレメトリの勾配情報によりゲーム内の目標速度が低下し、車が遅くなります。平地と同じ速度を保つには、ローラーをより速く回して（ケイデンスを上げて）高いパワーを捻出する必要があります。
- **下り坂**: 足を止めても（0W）、テレメトリの下り勾配を検知し、重力シミュレーションにより車がスルスルと自動滑走します。

つまり、ペダルの物理的な「重さ」こそ変化しないものの、「出しているパワーに対するスピードの出方（物理演算）」はスマートローラー使用時と全く同じようにテレメトリを活用して忠実にシミュレートされます。

## 2. センサーの組み合わせと適用方法

本アプリケーションで車を走らせるために必要なコアデータは「パワー（W）」です。センサーの組み合わせに応じて、いかにしてパワー値を算出し、ゲーム内の車速・アクセルに反映させるかを以下に定義します。

### パターンA: パワーメーターのみ（推奨）
- **取得データ**: パワー（W）
- **適用方法**: 最も正確な構成です。取得したパワー値をそのまま本アプリケーションの入力とします。
  - **シミュレーションモード時**: パワー値から自転車物理モデルを用いて目標速度を算出し、Forza内の車がその速度に追従するようPID制御でアクセルを操作します。
  - **アーケードモード時**: パワー値を基準FTPで割り、直接車のアクセル開度（0〜100%）にマッピングします。

### パターンB: スピードセンサーのみ（仮想パワー・zPowerの算出）
- **取得データ**: 後輪の回転速度（km/h）
- **適用方法**: 固定ローラーの負荷特性曲線（パワーカーブ）とスピードセンサーの速度データから、「仮想パワー（Virtual Power）」を算出します。算出された仮想パワーを本システムの入力（W）として扱います。
- **課題**: 固定ローラーの機種ごとにパワーカーブが異なるため、汎用的な曲線を適用するか、代表的なローラーのプロファイルを選択できる仕組みが必要です。

### パターンC: パワーメーター ＋ ケイデンスセンサー
- **取得データ**: パワー（W）、ケイデンス（RPM）
- **適用方法**: パワー値はパターンAと同様に車のアクセル制御に使用します。本アプリケーション（レースゲーム連動）においてケイデンスは車両制御に直接影響しませんが、将来的なUI拡張（画面上に現在のケイデンスを表示するなど）やログ出力のためにデータを保持します。

### パターンD: スピードセンサー ＋ ケイデンスセンサー
- **取得データ**: 速度（km/h）、ケイデンス（RPM）
- **適用方法**: スピードセンサーから仮想パワーを算出し（パターンBと同様）、車のアクセル制御に利用します。ケイデンスデータはUI表示等の参考値とします。

### パターンE: スピードセンサー ＋ パワーメーター (＋ ケイデンスセンサー)
- **取得データ**: パワー（W）、速度（km/h）
- **適用方法**: パワーメーターのデータ（実測値）を最優先で車両制御に使用します。スピードセンサーによる仮想パワー計算は行わず、速度データは無視するか参考値にとどめます。

## 3. 接続デバイス選択のためのCLI仕様案

各種BLE/ANT+センサー（パワー、スピード、ケイデンス）を個別に受信するためには、既存の「スマートローラー1台に自動接続する」ロジックから拡張し、接続するデバイスを選択・管理できるCLI（コマンドラインインターフェース）が必要です。

### 3.1. CLIの起動オプション
アプリケーション起動時に、使用するセンサーのMACアドレス（BLE）またはデバイスID（ANT+）を指定できるようにします。

```bash
# 対話形式で各種センサーの初期セットアップを行う
$ horizon-cycling --setup-sensorsq

# 引数で直接センサーを指定して起動（パワーメーターとケイデンスセンサーの組み合わせ）
$ horizon-cycling --power 00:11:22:33:44:55 --cadence 66:77:88:99:AA:BB

# スピードセンサーから仮想パワーを生成する場合
$ horizon-cycling --speed AA:BB:CC:DD:EE:FF --virtual-power-curve fluid-trainer-v1
```

### 3.2. 対話型セットアップのフロー（CLI）
1. **スキャン開始**: `Scanning for BLE/ANT+ devices... (10s)`
2. **デバイスリストの表示**:
   ```
   [1] Power Meter (00:11:22:33:44:55)
   [2] Speed Sensor (AA:BB:CC:DD:EE:FF)
   [3] Cadence Sensor (66:77:88:99:AA:BB)
   [4] Smart Trainer (11:22:33:44:55:66)
   ```
3. **パワーソースの選択**:
   `Select device for POWER (Enter number, or 0 to skip): 1`
4. **スピードソースの選択**:
   `Select device for SPEED (Enter number, or 0 to skip): 0`
   （スピードセンサーが選択され、かつパワーソースがない場合は、後続プロンプトで仮想パワー用プロファイルを選択させる）
5. **ケイデンスソースの選択**:
   `Select device for CADENCE (Enter number, or 0 to skip): 3`
6. **設定の保存**:
   `Configuration saved to config.json. Starting application...`

### 3.3. 設定の永続化と自動フォールバック
選択されたセンサー構成およびデバイスIDは `config.json` 等に保存されます。アプリケーションの起動フローは以下のようになります。

- **コンフィグが存在する場合**: 引数なしで `$ horizon-cycling` を起動すると、自動的に `config.json` を読み込み、登録されたセンサー構成での同時接続を試みます。
- **コンフィグが存在しない場合（初回起動時）**: 引数なしで起動した際、`config.json` が見つからなかった場合は、自動的に対話型のセットアップモード（`--setup-sensors` 実行時と同じ処理）が立ち上がり、ユーザーに初期デバイスの選択と登録を促します。

**保存後にデバイスを変更する（再設定する）場合の手順**:
一度コンフィグに保存されたデバイス構成を変更したい場合は、以下のいずれかの方法で行います。

1. **セットアッププロンプトの再実行（推奨）**
   再度 `$ horizon-cycling --setup-sensors` コマンドを実行して起動します。改めてデバイススキャンと対話プロンプトが開始され、新しく選択したセンサー情報で `config.json` が上書き更新されます。
   
2. **引数による一時的な上書き・変更**
   コンフィグに保存された設定を保持したまま、今回だけ別のセンサーを使いたい場合（あるいは特定の一部だけ置き換えたい場合）は、引数で直接指定して起動します。
   例: `$ horizon-cycling --power <新しいMACアドレス>`
   
3. **設定ファイルの直接編集**
   テキストエディタ等で直接 `config.json` を開き、記載されているデバイスIDやMACアドレス、仮想パワーカーブのプロファイル名などを手動で書き換えることも可能です。
